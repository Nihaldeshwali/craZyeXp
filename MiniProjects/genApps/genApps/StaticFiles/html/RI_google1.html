Let's consider a scenario where a price list of a product is given to you. It is basically a list of integer, indicate the price of the product on that day. You want to buy some item and will sell on higher price to get the maximum profit. How will buy and sell the item such that you can gain maximum profit for the following cases:
  • At most one buy-one sell is permitted.
  • We can have almost two buy - shell
  • Any number of buy-sell is allowed

Work out some examples !

Let consider following price list : {100, 180, 260, 310, 40, 535, 695}, where 100 is the day1 price, 180 for DAY2 and so on..If you but on day1 and sell on Day7 the profit will be 695-100 = 595. The Max profit would be when you buy at price 40 and sell at price 695, you will get max profit as 655.
  • Single Sell: We can have one Buy -> Sell sequenec and Profit is 695-40 = 655
  • Atmost two buy and sell: We have a sequence like : Buy->Sell->buy->Sell : Ans would be (310-100)+(695-40)
  • Unbound Buy and Sell: same as two

Formulate the problem

Let's consider the given array is A[1..N]. We need to find out two index i and j such that Max(A[j] > A[i]) Where j > i. We will buy the product at Day-i and we will sell at Day-j.
Similarly for 2nd version of the problem we need to find out 4 index i,j,k,l such that j > i and k > l which maximize (A[j]-A[i])+(A[k]-A[l]). The solution indicates We buy the first product at Day-i and Sell on Day-j. For second product , we can buy on Day-l, and Sell at Day-k.
In general sityuation, we we allow any number of buy and sell, we need to find out all pair of index (b1,s1),(b2,s2)..(bn,sn) such that si >bi which maximize the value of Sum(A[si]-A[bi]).

Solution 1: Two Loop Approach

The solution is as simple as below: Pick up every elemnt in the arry and compare with other elemnt, right on it and keep track the max diff so far.
Input : A[1..N] Output: MaxDiff Step1. initilize max_diff := 0 Step2. For i :=1 to N: Step 1.1. For j = i+1 to N and A[j] > A[i]: if (A[j] -A[i]) > max_diff and then update max_diff.

Derive the Complexity!

What is the time complexity of this problem ? O(n^2) - This is because we picked up each element and compare with all the elements in right side of this. thus For first element we need N comparism , For second elemnt it need n-1 comparisn and so on. Thus total comparrism require is N+(N-1)+(N-2)+...+1 = O(n^2)
What is the space complexity for this problem? It's O(1), As we are not using any space other than loop veraibles ( Which takes constact space), which depends on input size.
